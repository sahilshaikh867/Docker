## 1. Docker Engine âš™ï¸

Docker Engine is the core component that makes Docker work. It runs on the host system and is responsible for building images, running containers, managing networks, and handling storage.

It consists of three main parts:
- The Docker daemon, which performs all heavy tasks in the background
- The Docker API, which allows communication between tools and the daemon
- The Docker CLI, which users interact with using commands

Without Docker Engine, containers cannot be built or executed.

## 2. Docker Image ğŸ§±

A Docker image is a **read-only template** used to create containers. It contains the application code, runtime, libraries, and configuration required to run the application.

Images are built in layers, which makes them efficient and reusable. When a change is made, only the modified layer is rebuilt. Images are immutable, meaning they do not change once created.

In real projects, images represent application versions and are stored in registries for reuse and deployment.

A Docker container is a **running instance of an image**. It is where the application actually runs.

Containers are isolated from each other and from the host system, but they share the hostâ€™s kernel. This design makes containers lightweight and fast compared to virtual machines.

Containers can be started, stopped, restarted, and removed easily, which makes them ideal for dynamic and scalable systems.

## 4. Dockerfile ğŸ“„

A Dockerfile is a text file that defines how a Docker image is built. It contains step-by-step instructions such as selecting a base image, installing dependencies, copying application files, and defining the startup command.

The Dockerfile ensures that the image build process is automated and reproducible. Any engineer using the same Dockerfile will generate the same image.

In production environments, writing clean and optimized Dockerfiles is a critical skill.

## 5. Docker Registry ğŸ—‚ï¸

A Docker registry is a storage system for Docker images. It allows images to be shared between developers, CI/CD pipelines, and production environments.

Public registries like Docker Hub and private registries like AWS ECR are commonly used in industry. Registries act as a central source of truth for application images.

## 6. Docker Volumes ğŸ’¾

Docker volumes are used to store data generated by containers. Containers themselves are ephemeral, meaning data inside them is lost when they are removed.

Volumes allow data to persist independently of container lifecycles. They are commonly used for databases, logs, and configuration files that must survive container restarts.

Docker provides built-in networking to allow containers to communicate with each other and with external systems.

Containers can be attached to different network types depending on the use case. Proper networking configuration is essential for microservices, APIs, and distributed systems.
